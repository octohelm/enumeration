# 作用域

本文件作用于仓库根目录及其所有子目录；若子目录存在更近的 `AGENTS.md`，以更近规则覆盖冲突项。

# 规则优先级

1. 仅 `AGENTS.md` 中落盘的规则视为仓库协作规则。
2. 更近目录的 `AGENTS.md` 优先于上层同名约束。
3. 未被下层重写的上层规则继续继承。

# 变更边界

1. 先做最小盘点，再开始改动。
2. 只修改与当前任务直接相关的仓库内文件。
3. 不做无关功能扩展、顺手重构或批量风格清理。
4. 不因目录已存在而默认新增模块、执行入口或注册关系。
5. 不回滚、覆盖或整理自己未引入的变更，除非得到明确确认。

# 停止条件

1. 需求边界不清且存在多种互斥实现时，先暂停并确认。
2. 需要新增模块、入口、注册关系、workspace 边界或执行面模式时，先暂停并确认。
3. 发现项目结构、模块分层、入口组织或控制面职责异常时，先说明问题，再给出可执行建议。

# 人工接管条件

1. 需要执行高风险操作、破坏性命令或历史改写时，由人工接管。
2. 需要覆盖他人未确认的现有改动时，由人工决定。
3. 当前仓库事实不足以裁决结构调整方案时，由人工确认目标边界。

# 文件边界

1. `AGENTS.md` 只承载协作约束，不写项目背景、模块职责正文或命令细节。
2. `README.md` 只承载最小概述、职责边界和导航索引。
3. `justfile` 只承载统一执行入口，不混入协作规则或背景说明。

# 基本工程约束

1. 能完成最小相关验证时，必须先验证再交付。
2. 无法完成最小验证时，必须明确说明缺口、原因与风险。
3. 优先保持现有架构分层，不为局部便利破坏长期一致性。
4. 优先通过参数或依赖注入传递上下文，避免可变全局状态。
5. 避免过度工程、过早抽象和无必要注释。
6. 尽量使用中文，除非专用名称，包括但不限于文档、注释、错误和日志信息。

# 规则启用机制

只有写入 `AGENTS.md` 的规则才作为本仓库协作约束生效；对话中的临时偏好或未落盘说明，不自动成为长期规则。
